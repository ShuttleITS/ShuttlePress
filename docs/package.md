# 主题打包
## [获取主题包](https://pac.xiamii.com)
> 该打包机面向购买用户，用于在线生成主题包。首次登录后需绑定机场名称并填写配置，生成的前端包会锁定到用户的机场信息。

## 登录与机场名称锁定

- 首次登录打包机，在主页输入机场名称后即锁定，用户不可再次修改。
- 前端构建页面会从已绑定的机场名称中读取该字段，无需重复输入。

## 构建表单字段

- **站点 Logo**：支持 `public/` 目录下的文件路径（如 `/logo.png`）或外部 URL。
- **前端域名**：用于约束前端访问，最多 4 个，以逗号分隔，需填写完整域名（含协议可选，但域名必须完整）。
- **后端 API 地址**：指向机场后端，默认 `/api/v1/`，根据实际后端路径修改。
- **Apple Id Share（苹果账户分享）**：如需开启，先打开 toggle，再填写 `Apple Id Share API 地址`（默认 `/idhub-api/`）和 `IDHub API Key`（必填）。
- **客户端下载链接**：填写 OSS 或下载直链，供客户下载客户端。

## 打包流程

1) 登录打包机，首次输入机场名称（锁定）。
2) 在构建页填写/确认站点 Logo、前端域名、后端 API、Apple Id Share（可选）、客户端下载链接。
3) 点击「开始构建」后，会跳转到主页的构建队列等待；构建完成后自动跳转到构建下载页面，下载已构建的主题包（React+TS 静态前端）。
4) 将下载的主题包部署到你的前端域名。

## 注意事项

- 机场名称一旦锁定不可修改，如需调整请联系管理员重置。
- 前端域名必须为完整域名，最多 4 个，逗号分隔；域名外的空格请去除。
- 默认 API 路径 `/api/v1/` 和 `/idhub-api/` 可按后端路由实际情况调整。
- 如需前后端分离部署，后端 API 地址与 IDHub API 地址请保持默认路径（`/api/v1/`、`/idhub-api/`），避免因路径差异导致跨域或路由 404。
- Logo 可用本地静态文件或在线链接，确保打包后路径可访问。
- 额度限制：每个用户每日 2 次构建、5 次下载，合理安排构建次数，避免超额。
